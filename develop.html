INTRODUCTION

Welcome to the PWNHS Development Kit. This page covers basic requirements and features of the PWNHS website project.

What the PWNHS Development Kit is

To open up more avenues for engagement with the PWNHS community as a whole, we have opened up the PWNHS site project for tweaking by those in our organisation who are particularly code-savvy. The Development Kit provides basic structure for how the site should behave and establishes a path for anyone to help improve our website.

What the PWNHS Development Kit is not

This document does not hold proprietary APIs, Content Management Systems, of FTP clients. Developers are given access to static infastructure, not server-side technologies.

GETTING STARTED

Project Base

PWNHS is a GitHub Pages project hosted (consequently) on GitHub (pwnhs/pwnhs.github.io). You can visit the project here. To explore the project and offer contributions at a later time, you must first fork and clone the PWNHS repository.

Forking

To fork the project, just click the button below this block. You will be asked to fork the repo to your account, and that's it!

Fork

Cloning to Local

In order to be able to edit your cloned PWNHS repo, you will have to clone the fork to your local machine, where you can directly access and edit all the code of the site. Cloning can be done in two ways - via GitHub's desktop application or via command line.

Cloning via Desktop App

Cloning via GitHub's desktop app is a one-click process. After downloading the app, simply go to your forked repository of PWNHS (https://github.com/USER/pwnhs.github.io) and press Open in Desktop under the Clone or download button. This automates the entire process, and we suggest all but the most experienced take this route.

Cloning via Command Line

To create a local clone via command line, open up Terminal or Git Bash and type in git clone followed by the URL of your fork. You can find this URL by going to your fork of PWNHS and clicking the Clone or download button and copying the URL found below the Clone with HTTPS block. Your command line will look like this, with your GitHub username instead of USER: 

git clone https://github.com/USER/pwnhs.github.io.git
After you press Enter, your local clone should be created. Check to make sure all objects are unpacked and the command line reads done.

To allow for upstreaming (pulling of changes from the original repo to your fork), navigate to the location of your fork in the command line and type in git remote add upstream followed by the URL of the original PWNHS repo. Your command line will look like this: 
git remote add upstream https://github.com/pwnhs/pwnhs.github.io.git

Note

IMPORTANT NOTE: In order to submit pull requests to the Master, you must both fork and then clone the Master PWNHS repo. Only cloning it will not create a server-side branch of the repo, thereby not tying your clone to the Master and disallowing your ability to merge with the Master branch. More information on forking a repo here.
